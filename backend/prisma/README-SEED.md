# 🚀 OpenMarket 데이터베이스 시드 시스템

완전한 한국어 더미데이터로 데이터베이스를 구축하는 자동화 시스템입니다.

## 📋 개요

앱 심사를 통과할 수 있을 정도의 완전한 실제 서비스 수준 데이터를 생성합니다:
- **3,000개 이상**의 레코드
- **100% 한국어** 콘텐츠
- **모든 기능 시나리오** 포함

## 🎯 생성되는 데이터

### 👤 사용자 관련 (60개)
- 판매자 10개 (한국 업체명, 사업자정보)
- 사용자 25개 (한국 이름, 주소)
- 배송지 주소 25개

### 📦 상품 시스템 (1,800개+)
- 전체 상품 180개
- **옵션상품**: 색상/사이즈 옵션과 SKU 시스템
- 상품이미지 600개+
- 상품정보고시 360개+

### ⭐ 리뷰 시스템 (1,000개+)
- 리뷰 1,000개 이상
- 리뷰 이미지 포함

### 🛒 쇼핑 시스템 (200개+)
- 장바구니 15개
- 주문 80개 (다양한 배송상태)
- 쿠폰 시스템 완비

### 💬 고객지원 (150개+)
- 유저가이드 (상세한 한국어 가이드)
- 에러리포트 50개
- 문의/공지/FAQ 완비

### 📎 파일 시스템 (1,000개+)
- 첨부파일 1,000개 이상
- 배너 시스템

## 🚀 사용법

### 1. 완전한 DB 재구축 (상세 로그)
```bash
npm run rebuild-db
```
또는
```bash
bash prisma/rebuild-db.sh
```

### 2. 빠른 DB 재구축 (로그 최소화)
```bash
npm run quick-rebuild
```

### 3. 현재 DB 상태 확인
```bash
npm run check-db
```

### 4. Prisma Studio로 확인
```bash
npm run prisma:studio
```

## ⚡ 실행 순서

1. **DB 초기화**: 기존 데이터 완전 삭제 및 스키마 재생성
2. **기본 데이터**: 판매자, 사용자, 상품, 쿠폰 생성
3. **옵션 & 리뷰**: 옵션 시스템과 대량 리뷰 생성
4. **쇼핑 데이터**: 장바구니, 주문, 관심상품 생성
5. **지원 & 파일**: 고객지원 데이터와 첨부파일 생성

## 🎨 특징

### 🌟 현실적인 데이터
- 실제 한국 브랜드명과 제조사
- 자연스러운 한국어 리뷰와 문의
- 실제 쇼핑 패턴을 반영한 주문 데이터

### 🔧 완전한 기능 커버
- 단일상품 + 옵션상품 (색상/사이즈)
- 장바구니 → 주문 → 배송 플로우
- 쿠폰 발급 및 사용
- 리뷰 작성 및 이미지 첨부
- 고객지원 (문의/FAQ/에러리포트)

### 📊 앱 심사 대응
- 충분한 데이터 볼륨
- 모든 예외 상황 케이스 포함
- 실제 서비스 운영 수준의 완성도

## ⚠️ 주의사항

1. **기존 데이터 완전 삭제**: `rebuild-db`는 모든 데이터를 삭제합니다
2. **실행 시간**: 완전 재구축에 약 2-3분 소요
3. **네트워크**: 이미지 URL 생성을 위해 인터넷 연결 필요
